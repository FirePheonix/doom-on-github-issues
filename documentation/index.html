Fastest deploy path

Project

One GitHub issue is one Doom session. Comments become input ticks. The issue body becomes the current frame display.

Open repository

Current release

v4.0.0 focuses on lower latency, cached early menu states, deterministic frame publishing, and a persistent DoomGeneric worker path.

Deploy

Host it on Vercel

  1. Create a Vercel project from this repository.
  2. Set the project root to documentation/.
  3. Use the static or "Other" preset.
  4. Leave the build command empty.
  5. If Vercel asks for an output directory, use ..

If you deploy the repository root instead, the top-level vercel.json redirects / to /documentation/.

Quick summary

What v4 changed

User-facing

  • Early menu states can resolve from cached frames.
  • One comment can batch multiple commands into one final frame update.
  • The visible path avoids long replay stalls when a live worker is ready.

Runtime

  • The DoomGeneric worker stays alive and advances in memory.
  • Frame publishing uses deterministic S3 object naming for shared menu frames.
  • Background sync coalesces to the newest target instead of draining stale work.
